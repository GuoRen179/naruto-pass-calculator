# 项目形态建议

项目同时面向网页和微信小程序，建议采用“共享计算核心 + 多端界面”的结构。

## 目录结构

```text
apps/
  web/            # 网页端界面
  miniprogram/    # 微信小程序端界面
packages/
  core/           # 忍法帖计算规则、忍者数据、价格公式
docs/
  rules.md        # 已知规则原始整理
```

## 开发原则

- 计算公式只写在 `packages/core` 中。
- 网页和小程序只负责收集输入、展示结果。
- 新增忍者、调整价格、改赛季基础收益时，优先修改核心数据。
- 先做可用计算器，再补视觉、分享图、FAQ 和活动入口。

## 推荐实现顺序

1. 完成核心公式和测试。
2. 开发网页端，方便快速预览和调试。
3. 复用核心逻辑开发微信小程序端。
4. 增加常见方案预设，例如“已有 50 片补整卡”“只换秘卷到 5 级”。
5. 增加分享结果和新手解释文案。
